\doxysubsection{Detailed Description}
Configuration class for securing the Codingsy Employee Management System (EMS) in a production environment. This class defines the security filter chain, authentication mechanisms, and security-\/related beans specific to the production profile. 

This configuration enforces HTTPS, limits sessions to one per user, integrates CSRF protection with a custom filter, and configures authentication methods (form login and HTTP basic). It is activated only when the {\ttfamily prod} profile is active. 

The configuration relies on customizers ({\ttfamily \doxylink{interfacecom_1_1codingsy_1_1ems_1_1security_1_1config_1_1customizer_1_1_default_customizer}{Default\+Customizer}}) to modularize and extend security settings for various aspects such as session management, CORS, CSRF, authorization, and exception handling.

Defines the default security filter chain for production environment. 

Configures the security settings including session management (max one session per user), channel security (enforces HTTPS), CORS, CSRF protection with a custom {\ttfamily \doxylink{classcom_1_1codingsy_1_1ems_1_1security_1_1filter_1_1_csrf_cookie_filter}{Csrf\+Cookie\+Filter}}, authorization rules, form login, HTTP basic authentication, and exception handling. If a new login is attempted, the existing session is invalidated, and the user is redirected to the {\ttfamily /invalid\+Session} page.
